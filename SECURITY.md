# 보안 정책

## 지원 범위

보안 수정은 현재 `main` 브랜치에 적용합니다. 별도 버전 브랜치는 지원하지 않습니다.

## 취약점 제보

GitHub 저장소의 **Security → Report a vulnerability** 기능으로 비공개 제보해 주세요. 공개 Issue, Discussion, Pull Request에는 취약점의 재현 절차, 토큰, 비밀번호, 내부 주소와 개인정보를 올리지 마세요.

제보에는 영향 범위, 재현 조건, 예상 결과와 실제 결과를 포함해 주세요. 확인 후 영향과 수정 범위를 검토해 답변하겠습니다.

일반 버그와 기능 요청은 공개 Issue를 사용할 수 있지만 실제 비밀값은 예시에도 포함하지 않아야 합니다.
